# TAVR Analytics - 模块一实现

## 概述

这是TAVR-Analytics Workflow 3D Slicer插件的模块一实现，专门用于**数据导入与场景准备**。根据详细设计文档，该模块实现了完整的4D心脏CT数据加载、患者信息配置、瓣膜选择和心动周期管理功能。

## 主要功能

### 1. 数据导入与验证
- **4D DICOM序列加载**：支持从DICOM浏览器加载4D心脏CT序列
- **自动序列检测**：智能检测场景中已有的序列数据
- **多容积组合**：支持将多个容积节点组合成4D序列
- **数据验证**：确保加载的数据是有效的4D心脏CT序列

### 2. 患者信息管理
- **DICOM元数据解析**：自动从DICOM头文件提取患者基本信息
- **完整信息收集**：对应"杭州方案术后CT核心实验室评估表"的所有字段
- **必填项验证**：强制要求关键信息（受试者编号、瓣膜信息）

### 3. 瓣膜特异性配置
- **动态瓣膜选择**：基于JSON配置文件的瓣膜品牌和型号选择
- **策略模式支持**：为不同瓣膜型号的特异性测量算法提供配置基础
- **可扩展性**：通过修改配置文件轻松添加新的瓣膜型号

### 4. 心动周期管理
- **交互式时间轴**：滑块控制，支持流畅的心动周期导航
- **实时视图同步**：与Slicer的2D/3D视图实时同步
- **关键时相标记**：精确标记舒张末期和收缩末期
- **时相信息持久化**：保存帧索引和R-R间期百分比

## 核心数据结构

### TAVRStudySession（单例模式）
- 中央化的状态管理对象
- 持有所有关键数据和节点引用
- 提供标准化的API供其他模块调用

### PatientData（数据类）
- 完整的患者信息封装
- 直接对应评估表单的数据结构
- 支持DICOM自动填充和手动编辑

## 界面设计

### 主界面
- **状态显示区域**：显示当前配置状态
- **数据导入按钮**：启动数据加载对话框
- **演示数据按钮**：快速加载测试数据
- **心动周期管理面板**：时相导航和标记功能

### 数据加载对话框
- **数据加载区域**：DICOM序列导入
- **患者信息表单**：完整的信息收集界面
- **瓣膜选择器**：级联下拉菜单
- **验证反馈**：实时的必填项检查

## 使用方法

### 1. 基本工作流
1. 打开TAVR Analytics模块
2. 点击"数据导入与配置"按钮
3. 在对话框中加载4D DICOM序列
4. 填写/确认患者信息
5. 选择瓣膜品牌和型号
6. 点击"确认并继续"
7. 使用心动周期管理功能

### 2. 快速测试
1. 点击"加载演示数据"按钮
2. 系统自动创建模拟的4D序列数据
3. 直接进入心动周期管理界面

### 3. 心动周期标记
1. 使用滑块浏览心动周期
2. 找到舒张末期时点，点击"标记舒张末期"
3. 找到收缩末期时点，点击"标记收缩末期"
4. 标记信息自动保存并显示

## API接口

### TAVRStudySession公共方法
- `get_patient_data()` - 获取患者数据
- `get_volume_sequence_node()` - 获取4D序列节点
- `get_sequence_browser_node()` - 获取序列浏览器节点
- `get_selected_valve()` - 获取选择的瓣膜信息
- `get_marked_phase(phase_name)` - 获取标记的时相信息
- `is_ready()` - 检查模块是否准备完成
